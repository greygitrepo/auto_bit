# Paper Trading (페이퍼 트레이딩)

## 개요

Paper Trading은 **메인넷 실시간 데이터**를 사용하되, 실제 주문 대신 **가상 주문 엔진**으로 거래를 시뮬레이션하는 모드이다.

### 왜 테스트넷이 아닌가?

| 항목 | 테스트넷 | 메인넷 페이퍼 |
|------|---------|-------------|
| 가격 데이터 | 실제와 다름 | 실제 데이터 |
| 유동성 | 비현실적 | 실제 반영 가능 |
| 오더북 | 비현실적 | 실제 데이터 참조 |
| 펀딩비 | 실제와 다름 | 실제 데이터 |
| 거래 쌍 | 제한적 | 전체 지원 |
| API 안정성 | 불안정할 수 있음 | 안정적 |

메인넷 페이퍼 트레이딩은 실전과 동일한 시장 조건에서 전략을 검증할 수 있다.

---

## 동작 방식

```
[Bybit Mainnet API] ──── 실시간 데이터 ────→ [전략 파이프라인]
                                                     │
                                              매매 신호 생성
                                                     │
                                                     ▼
                                          ┌──────────────────┐
                                          │  Paper Executor   │
                                          │                  │
                                          │  가상 잔고 관리   │
                                          │  가상 체결 엔진   │
                                          │  수수료 시뮬레이션 │
                                          │  P&L 계산        │
                                          └──────────────────┘
                                                     │
                                                     ▼
                                          [DB: paper_trades]
```

### 메인넷 API 키 요구사항

Paper 모드에서도 메인넷 API 키가 필요하다:
- **필요 권한:** 읽기 전용 (시세 조회)
- **불필요 권한:** 거래, 출금
- 읽기 전용 키만으로도 WebSocket 시세 데이터 수신 가능

## 가상 체결 엔진

### 시장가 주문

```
체결가 = 현재가 ± 슬리피지
슬리피지 = 현재가 × slippage_bps / 10000

예: BTC 65000 USDT, slippage_bps=5
  → 매수 체결가: 65000 × (1 + 5/10000) = 65032.5
  → 매도 체결가: 65000 × (1 - 5/10000) = 64967.5
```

### 지정가 주문

```
미체결 큐에 추가 → 매 틱/캔들마다 확인:
  매수: 현재 저가 ≤ 지정가 → 체결
  매도: 현재 고가 ≥ 지정가 → 체결
```

### SL/TP 주문

```
매 틱/캔들마다 확인:
  LONG 포지션:
    저가 ≤ SL가 → 손절 체결 (SL가로 체결, 슬리피지 적용 가능)
    고가 ≥ TP가 → 익절 체결

  SHORT 포지션:
    고가 ≥ SL가 → 손절 체결
    저가 ≤ TP가 → 익절 체결
```

### 수수료

```
수수료 = 체결금액 × fee_rate

시장가(Taker): 0.06%
지정가(Maker): 0.01%
```

### 펀딩비

```
8시간마다 (00:00, 08:00, 16:00 UTC):
  펀딩비용 = 포지션크기 × 현재가 × 실제펀딩비율
  LONG: 잔고 -= 펀딩비용 (펀딩비 양수일 때)
  SHORT: 잔고 += 펀딩비용 (펀딩비 양수일 때)
```

## Paper → Live 전환 체크리스트

Paper 모드에서 충분히 검증한 후 Live로 전환:

### 최소 검증 기준 (권장)

- [ ] 최소 2주간 paper 트레이딩 운영
- [ ] 100회 이상 거래 샘플
- [ ] 양(+)의 수익률 확인
- [ ] 최대 드로다운이 설정 한도 이내
- [ ] 승률, 손익비 등 성과 지표 안정적
- [ ] 에러/장애 없이 안정적 운영 확인

### 전환 절차

1. `config/app.yaml`에서 `mode: paper` → `mode: live` 변경
2. `config/credentials.yaml`에서 거래 권한이 있는 API 키 설정
3. **소액으로 시작** — `asset.yaml`에서 `risk_per_trade_pct`를 낮게 설정
4. 모니터링 강화 — 텔레그램 알림 전체 활성화
5. 점진적으로 자본 규모 확대

## 성과 비교

Paper와 Live의 성과를 비교하여 시뮬레이션의 정확도를 검증:

```
paper_trades vs live_trades 비교:
- 같은 기간, 같은 전략의 신호 일치율
- 체결가 차이 (슬리피지 정확도)
- 수익률 차이
```

향후 이 데이터를 기반으로 paper 엔진의 슬리피지/수수료 파라미터를 보정할 수 있다.
