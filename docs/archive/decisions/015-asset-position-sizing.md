# ADR-015: 포지션 사이징 및 레버리지 방식

## 상태

`결정됨`

## 날짜

2026-03-25

## 맥락

종목당 전체 자본의 5%를 투입하기로 결정했다 (ADR-010).
구체적인 포지션 크기 계산 방식과 레버리지 결정 방식을 정해야 한다.

조건:
- 단타 (최대 90분 보유)
- 신규상장 종목 (변동성 큼)
- SL: ATR(14) × 1.5 (0.3~2.0%)
- 최대 3 포지션 동시 운영

## 결정

### 포지션 사이징: 리스크 기반 역산 + 5% 캡

"거래당 리스크 금액 고정" 방식을 채택하되, 투입 자본이 5%를 넘지 않도록 상한을 건다.

```
거래당 리스크 = 전체 자본 × 1% (고정)

포지션 크기 계산:
  risk_amount = balance × 0.01
  sl_distance_pct = ATR 기반 SL 거리 (%)
  position_size = risk_amount / sl_distance_pct

  max_position = balance × 0.05
  position_size = min(position_size, max_position)

  leverage = position_size / margin
  leverage = min(leverage, max_leverage)
```

왜 리스크 기반인가:
- 고정 금액(5% 단순 투입) 방식은 SL 거리에 따라 실제 리스크가 0.15%~1.0%로 들쭉날쭉
- 리스크 기반이면 **매 거래에서 잃을 수 있는 최대 금액이 동일** (자본의 1%)
- 변동성이 큰 신규상장 종목에서 특히 중요 — ATR이 큰 종목은 자동으로 작은 포지션

왜 1%인가 (2%가 아닌):
- 단타는 거래 빈도가 높아 연속 손절 가능성 있음
- 1% × 3 포지션 동시 손절 = 3% → 감내 가능
- 1% × 일일 10회 전부 손절 = 10% → 드로다운 경고 수준 → 자동 중단
- 2%면 같은 시나리오에서 20% → 과도

### 레버리지: SL 기반 동적 결정 (상한 제한)

레버리지를 고정하지 않고, SL 거리에 따라 자동 계산한다.

```
SL 거리    필요 레버리지 (1% 리스크, 5% 마진 기준)    실제 적용
──────────────────────────────────────────────────────────
0.3%       ~3.3x                                      3x
0.5%       ~2.0x                                      2x
1.0%       ~1.0x                                      1x
1.5%       ~0.67x → 포지션 축소                       1x (크기 축소)
2.0%       ~0.50x → 포지션 축소                       1x (크기 축소)
```

규칙:
- 최소 레버리지: 1x (레버리지 없는 것과 동일)
- 최대 레버리지: 5x
- SL이 넓으면 레버리지를 낮추는 것이 아니라 포지션 크기를 줄여서 리스크 유지

### 계산 예시

```
자본: 10,000 USDT
종목: NEWCOINUSDT, 현재가 1.00 USDT
ATR(14): 0.012 (1.2%)
SL 거리: 0.012 × 1.5 = 0.018 (1.8%)

risk_amount = 10,000 × 0.01 = 100 USDT
position_size = 100 / 0.018 = 5,555 USDT
max_position = 10,000 × 0.05 = 500 USDT
→ position_size = min(5,555, 500) = 500 USDT (5% 캡 적용)

실제 리스크 = 500 × 0.018 = 9 USDT (0.09%) ← 1%보다 작음, 안전
leverage = 1x (500 USDT 마진으로 500 USDT 포지션)
```

```
자본: 10,000 USDT
종목: ANOTHERCOINUSDT, 현재가 0.50 USDT
ATR(14): 0.002 (0.4%)
SL 거리: 0.002 × 1.5 = 0.003 (0.6%) → 최소 SL 0.3% 이상이므로 OK

risk_amount = 10,000 × 0.01 = 100 USDT
position_size = 100 / 0.006 = 16,666 USDT
max_position = 10,000 × 0.05 = 500 USDT
→ position_size = 500 USDT (5% 캡 적용)

실제 리스크 = 500 × 0.006 = 3 USDT (0.03%)
leverage = 1x
```

5% 캡이 대부분의 경우 먼저 적용되어 실제 리스크는 1%보다 훨씬 작아진다. 이는 보수적인 운영이며, 단타 + 신규상장 조합에 적합하다.

### 설정

```yaml
# config/strategy/asset.yaml
strategies:
  fixed_ratio:
    # 포지션 사이징
    capital_per_position_pct: 5.0    # 종목당 최대 투입 (캡)
    risk_per_trade_pct: 1.0          # 거래당 리스크 (자본 대비)

    # 레버리지
    min_leverage: 1
    max_leverage: 5

    # 동시 포지션
    max_concurrent_positions: 3
    max_per_symbol: 1
```

## 영향

- 레버리지가 동적이므로 주문 시 레버리지 설정 API 호출 필요
- 5% 캡으로 인해 대부분 1~2x 레버리지로 운영될 가능성 높음
- 실제 리스크가 1%보다 작아지는 경우가 많아 자본 보존에 유리
- Paper 엔진에서도 동일 로직 적용
