# ADR-003: 기술적 지표 라이브러리 선택

## 상태

`결정됨`

## 날짜

2026-03-25

## 맥락

EMA, RSI, MACD 등 기술적 지표 계산이 필요하다.
직접 구현하거나 외부 라이브러리를 사용할 수 있다.

## 선택지

### 선택지 A: pandas-ta

**설명:**
pandas DataFrame 기반 기술적 분석 라이브러리.

**장점:**
- 130+ 지표 지원
- pandas와 자연스러운 통합 (`df.ta.ema()`)
- 순수 Python, 설치 간편
- 활발한 유지보수

**단점:**
- TA-Lib 대비 일부 지표 계산 속도 느림
- 대량 데이터 시 메모리 사용량

### 선택지 B: TA-Lib (ta-lib)

**설명:**
C 기반 고성능 기술적 분석 라이브러리.

**장점:**
- 가장 빠른 계산 속도
- 업계 표준 (트레이딩뷰 등과 동일 결과)
- 200+ 지표 지원

**단점:**
- C 라이브러리 별도 설치 필요 (시스템 의존성)
- 설치 과정이 OS별로 까다로울 수 있음
- Python wrapper가 다소 오래됨

### 선택지 C: 자체 구현

**설명:**
numpy/pandas 기반으로 필요한 지표만 직접 구현.

**장점:**
- 외부 의존성 없음
- 정확한 로직 제어
- 필요한 것만 구현하여 가벼움

**단점:**
- 개발 시간 소요
- 엣지 케이스 처리 검증 필요
- 지표 추가 시마다 구현 필요

## 결정

**선택지 A: pandas-ta** 채택.

이유:
- pandas DataFrame과 자연스럽게 통합되어 데이터 파이프라인이 깔끔하다 (`df.ta.ema()`).
- 130+ 지표를 지원하여 향후 전략 확장에 충분하다.
- 순수 Python이므로 C 라이브러리 설치 없이 `pip install`만으로 설치 가능하다. 배포 환경(Linux/Mac/Windows)에 구애받지 않는다.
- TA-Lib 대비 속도가 느리지만, 200개 캔들 × 50개 심볼 수준의 데이터에서는 체감 차이가 없다. Strategy Engine이 독립 프로세스(P2)로 동작하므로 계산 시간이 다른 프로세스를 블로킹하지도 않는다.
- 자체 구현은 개발 시간 대비 이점이 적다.

## 영향

- 지원 지표: 130+ (EMA, RSI, MACD, Bollinger, ATR, OBV, VWAP 등 모두 포함)
- 설치: `pip install pandas-ta` (시스템 의존성 없음)
- 데이터 형식: pandas DataFrame 기반으로 통일
- 성능: 현재 규모에서 충분, P2 프로세스 격리로 블로킹 없음
