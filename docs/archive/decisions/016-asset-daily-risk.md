# ADR-016: 일일 리스크 한도 및 연속 손절 대응

## 상태

`결정됨`

## 날짜

2026-03-25

## 맥락

단타 거래는 하루에 거래 빈도가 높다.
3 포지션 × 최대 90분 = 4.5시간 사이클. 24시간 기준 최대 ~15회 거래 가능.
일일 리스크 한도, 거래 횟수 제한, 연속 손절 시 대응을 결정해야 한다.

## 결정

### 일일 거래 한도

```
일일 최대 거래: 15회

이유:
- 3 포지션 × 90분 = 4.5시간 사이클, 하루 ~5 사이클 가능
- 5 사이클 × 3 포지션 = 15회
- 이 이상은 과잉 거래(overtrading)
```

### 일일 손실 한도

```
일일 최대 손실: 전체 자본의 3%

이유:
- 거래당 리스크 1% × 3회 연속 손절 = 3% → 즉시 당일 거래 중단
- 실제로는 5% 캡으로 리스크가 1% 미만이므로 3% 도달하려면 상당수 손절 필요
- 3%는 충분히 보수적이면서도 거래 기회를 과도하게 제한하지 않음
```

### 연속 손절 쿨다운

연속으로 손절이 발생하면 시장 상태가 전략에 불리한 것이므로, 강제 대기한다.

```
연속 손절 대응:
  2회 연속 손절 → 30분 쿨다운 (경고 알림)
  3회 연속 손절 → 당일 거래 중단 (긴급 알림)

쿨다운 중:
  - 새 포지션 진입 불가
  - 기존 포지션은 정상 관리 (SL/TP/트레일링)
  - 쿨다운 해제 후 스캐너 재실행

연속 손절 카운터:
  - 익절/트레일링/전략 청산 발생 시 카운터 초기화
  - 시간 초과 청산은 손절로 간주하지 않음 (중립)
  - 날짜 변경(UTC 0시) 시 초기화
```

왜 이 기준인가:
- 단타에서 연속 2회 손절은 "시장이 전략에 맞지 않는 상태"를 시사
- 쿨다운 없이 계속 진입하면 감정적 거래와 동일한 패턴 (무한 손절 루프)
- 30분 대기 동안 시장 상태가 변할 수 있고, 스캐너가 다른 종목을 찾을 수 있음

### 일일 리셋 (UTC 0시)

```
매일 UTC 0시:
  - 일일 손실 카운터 초기화
  - 일일 거래 횟수 카운터 초기화
  - 연속 손절 카운터 초기화
  - 전일 성과 텔레그램 리포트 전송
```

### 설정

```yaml
# config/strategy/asset.yaml
daily_limits:
  max_daily_loss_pct: 3.0          # 일일 최대 손실 3%
  max_daily_trades: 15             # 일일 최대 거래 15회
  reset_time_utc: "00:00"         # 일일 리셋 시간

consecutive_loss:
  cooldown_after: 2                # N회 연속 손절 후 쿨다운
  cooldown_minutes: 30             # 쿨다운 시간
  stop_after: 3                    # N회 연속 손절 시 당일 중단
```

## 영향

- Order Manager: 일일 거래 카운터, 손실 카운터, 연속 손절 카운터 관리
- Notification: 쿨다운 진입/해제, 당일 중단 알림
- 성과 분석: 쿨다운/중단 발동 빈도 기록 → 전략 튜닝 참고
