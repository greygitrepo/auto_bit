# ADR-008: 스캐너 결과 변경 시 포지션 처리 방식

## 상태

`결정됨`

## 날짜

2026-03-25

## 맥락

Scanner는 주기적으로 거래 대상 종목을 재선정한다.
이전 스캔에서 선정되어 포지션이 열린 종목이, 다음 스캔에서 탈락할 수 있다.
이 경우 열린 포지션의 처리 방식을 결정해야 한다.

**예시 시나리오:**

```
스캔 T1: 선정 → [BTC, ETH, SOL, DOGE, XRP]
  → ETH LONG 진입, SOL SHORT 진입

스캔 T2: 선정 → [BTC, ETH, AVAX, LINK, ARB]
  → SOL 탈락 (SOL SHORT 포지션 보유 중)
  → DOGE, XRP 탈락
  → AVAX, LINK, ARB 신규 진입
```

SOL SHORT 포지션을 어떻게 할 것인가?

## 선택지

### 선택지 A: 포지션 유지 — SL/TP에 의한 자연 종료

**설명:**
스캐너 탈락과 무관하게, 열린 포지션은 SL/TP 또는 Position Strategy의 청산 신호로만 종료한다.
탈락 종목의 데이터 수집은 포지션이 열려있는 동안 계속 유지한다.

**장점:**
- 전략의 진입 근거가 유효한 동안 포지션 유지 (조기 청산 방지)
- SL/TP가 서버사이드에 있으므로 안전
- 스캐너와 포지션 전략의 관심사가 명확히 분리

**단점:**
- 탈락 종목의 데이터 수집을 계속 유지해야 함
- 포지션 유지 기간이 길어질 수 있음
- "관심 밖" 종목에 자본이 묶임

### 선택지 B: 즉시 청산

**설명:**
스캐너에서 탈락한 종목의 열린 포지션을 시장가로 즉시 청산한다.

**장점:**
- 자본을 빠르게 회수하여 새로운 기회에 투입
- 포지션 관리가 단순

**단점:**
- 수익 중인 포지션도 강제 청산될 수 있음
- 전략의 진입 근거와 무관한 청산 → 전략 성과 왜곡
- 불필요한 거래 비용(수수료) 발생

### 선택지 C: 조건부 처리

**설명:**
탈락 종목의 포지션 상태에 따라 다르게 처리한다.

```
탈락 종목의 포지션:
  수익 중 (unrealized PnL > 0):
    → 트레일링 스탑 활성화, 이익 보호하면서 유지
  손실 중 (unrealized PnL < 0):
    → SL까지 유지 (이미 리스크 관리 중)
  진입 직후 (보유 시간 < threshold):
    → 유지 (아직 판단하기 이름)
```

**장점:**
- 상황에 맞는 유연한 대응
- 수익 보호 + 손실 관리 균형

**단점:**
- 구현 복잡도 증가
- 조건 파라미터 튜닝 필요
- 예외 케이스 많아짐

## 결정

**선택지 C: 조건부 처리** 채택.

탈락 종목의 포지션 상태에 따라 다르게 처리한다.

```
탈락 종목의 포지션:
  수익 중 (unrealized PnL > 0):
    → 트레일링 스탑 활성화, 이익 보호하면서 유지
  손실 중 (unrealized PnL < 0):
    → SL까지 유지 (이미 리스크 관리 중)
  진입 직후 (보유 시간 < threshold):
    → 유지 (아직 판단하기 이름)
```

### 추가 고려사항

- 탈락 종목이라도 포지션이 열려있는 동안 Data Collector에서 해당 심볼의 WebSocket 구독을 유지한다.
- 포지션이 모두 청산되면 구독을 해제하고, 해당 슬롯이 비면 스캐너가 새 종목을 찾는다.
- 트레일링 스탑 파라미터는 `config/strategy/position.yaml`에서 관리한다.

## 영향

- 스캐너: 빈 슬롯이 생겨야 재실행 (포지션 종료 이벤트 트리거)
- Data Collector: 포지션 보유 중인 종목은 구독 유지
- Position Strategy: 탈락 종목에 대해 트레일링 스탑 모드 전환 로직 필요
- 최대 구독 심볼: BTC + ETH + 3개 포지션 = 5개 (가벼움)
