# ADR-005: 거래 유형 (현물 vs 선물)

## 상태

`결정됨`

## 날짜

2026-03-25

## 맥락

Bybit은 현물(Spot), USDT 무기한 선물(Linear), 인버스 선물(Inverse) 등 여러 거래 유형을 지원한다.
시스템이 어떤 거래 유형을 대상으로 할지 결정해야 한다.

## 선택지

### 선택지 A: USDT 무기한 선물 (Linear) 전용

**설명:**
USDT를 담보로 하는 무기한 선물 계약만 지원한다.

**장점:**
- 롱/숏 양방향 거래 가능
- 레버리지 활용 가능
- USDT 기준으로 손익 계산이 직관적
- 가장 유동성이 높은 시장
- 시스템 복잡도 낮음 (하나의 계좌 유형)

**단점:**
- 펀딩비 비용 발생
- 레버리지로 인한 청산 리스크
- 선물 거래 경험 필요

### 선택지 B: 현물(Spot) 전용

**설명:**
현물 시장에서만 거래한다.

**장점:**
- 청산 리스크 없음
- 단순한 구조
- 장기 보유 가능

**단점:**
- 롱만 가능 (하락장 대응 불가)
- 레버리지 불가 (자본 효율 낮음)
- 수익 기회 제한

### 선택지 C: 선물 + 현물 모두 지원

**설명:**
선물과 현물 시장을 모두 지원하여 전략에 따라 선택한다.

**장점:**
- 전략 다양성 극대화
- 시장 상황에 따른 유연한 대응

**단점:**
- 구현 복잡도 크게 증가
- 계좌 관리, 주문 로직 이중화
- 초기 단계에서 과한 범위

## 결정

**선택지 A: USDT 무기한 선물 (Linear) 전용** 채택.

이유:
- 롱/숏 양방향 거래로 상승장·하락장 모두 수익 기회를 잡을 수 있다.
- USDT 기준 손익 계산이 직관적이어서 자산관리 전략 구현이 단순하다.
- Bybit에서 가장 유동성이 높은 시장이므로 시장가 주문(ADR-004) 시 슬리피지가 최소화된다.
- 단일 계좌 유형으로 시스템 복잡도를 낮출 수 있다.
- 펀딩비와 청산 리스크는 자산관리 전략(Asset Strategy)에서 관리한다.

### API 사용
```python
# Bybit V5 API category 파라미터
category = "linear"  # 모든 API 호출에 고정
```

## 영향

- API: `category="linear"` 고정, USDT 무기한 선물 전용
- 포지션 관리: 마진 모드(Cross/Isolated), 레버리지, 청산가 계산 필요
- 자산관리: 펀딩비 비용 고려, 레버리지 리스크 관리 포함
- 수수료: Maker 0.01%, Taker 0.06% + 펀딩비(8시간마다)
- 스캐너: `linear` 카테고리 심볼만 대상
