# ADR-014: 포지션 청산 전략

## 상태

`결정됨`

## 날짜

2026-03-25

## 맥락

단타(최대 90분) 거래에서 청산 전략을 결정해야 한다.
청산은 수익 보호와 손실 제한의 균형이 핵심이다.

청산이 발생하는 경우:
1. 손절 (Stop Loss)
2. 익절 (Take Profit)
3. 트레일링 스탑
4. 전략 청산 신호
5. 시간 초과 (90분)
6. 스캐너 탈락 (ADR-008: 조건부 처리)

## 결정

### 청산 우선순위

```
1순위: SL/TP 서버사이드 주문 (Bybit에서 자동 체결)
2순위: 시간 초과 강제 청산 (90분)
3순위: 전략 청산 신호 (조건 충족 시)
4순위: 스캐너 탈락 시 조건부 처리 (ADR-008)
```

### 1. 손절 (Stop Loss)

**ATR 기반 동적 손절** 채택.

```
SL 거리 = ATR(14) × 1.5

LONG: SL = 진입가 - SL거리
SHORT: SL = 진입가 + SL거리

단, SL%가 아래 범위를 벗어나면 보정:
  최소: 0.3% (너무 타이트하면 노이즈에 청산)
  최대: 2.0% (단타에서 2% 이상 손실은 과다)
```

왜 ATR 기반인가:
- 신규상장 종목은 변동성이 종목마다 크게 다름
- 고정 %는 변동성이 큰 종목에서 너무 타이트하거나, 작은 종목에서 너무 느슨함
- ATR이 종목별 변동성을 자동 반영

서버사이드 조건부 주문으로 설정하여 클라이언트 장애 시에도 보호.

### 2. 익절 (Take Profit)

**리스크 리워드 비율 기반** 채택.

```
TP 거리 = SL 거리 × R:R 비율

기본 R:R = 2.0
  → SL이 1%면 TP는 2%
  → 승률 34% 이상이면 손익분기

LONG: TP = 진입가 + TP거리
SHORT: TP = 진입가 - TP거리
```

서버사이드 조건부 주문으로 설정.

### 3. 트레일링 스탑

수익 구간 진입 시 이익을 보호한다.

```
활성화 조건: 미실현 수익 ≥ SL거리 × 1.0 (= 1R 수익 도달)

활성화 후:
  트레일링 콜백 = ATR(14) × 0.8
  → 최고점(LONG) / 최저점(SHORT) 대비 콜백만큼 되돌리면 청산

예시 (LONG, 진입가 100, SL거리 1.5):
  가격 101.5 도달 → 트레일링 활성화
  가격 103.0까지 상승 → 트레일링 SL = 103.0 - 1.2 = 101.8
  가격 101.8로 하락 → 청산 (수익 +1.8%)
```

트레일링 스탑은 클라이언트(P3)에서 관리:
- Bybit 서버사이드 TP를 계속 업데이트하는 방식
- 원래 TP 도달 전에 수익을 확보하는 역할

### 4. 전략 청산 신호

진입 근거가 무너졌을 때 TP/SL 도달 전에 청산한다.

```
LONG 청산 조건 (OR):
  - EMA5가 EMA10 아래로 교차 (단기 추세 전환)
  - RSI가 75 초과 후 70 아래로 하락 (모멘텀 꺾임)
  - 거래량 급감 (직전 5캔들 평균의 0.3배 이하)

SHORT 청산 조건 (OR):
  - EMA5가 EMA10 위로 교차
  - RSI가 25 미만 후 30 위로 상승
  - 거래량 급감
```

### 5. 시간 초과 강제 청산

```
경과 시간  상태
─────────────────────
 0~75분    정상 운영
75분       텔레그램 경고 알림
90분       시장가 강제 청산 (무조건)
```

시간 초과 청산은 최종 안전장치. 다른 청산 조건이 먼저 발동되는 것이 정상.

### 6. 스캐너 탈락 시 (ADR-008 조건부 처리)

```
수익 중: 트레일링 스탑 강제 활성화 (미활성 상태였어도)
손실 중: SL까지 유지
진입 직후 (10분 이내): 유지
```

### 청산 흐름 정리

```
[매 5분봉 완성 시]
  │
  ├─ SL/TP 체결 확인 (서버사이드에서 이미 처리됐을 수 있음)
  │
  ├─ 시간 초과 확인 → 90분 초과 시 즉시 시장가 청산
  │
  ├─ 트레일링 스탑 업데이트 (활성 상태인 경우)
  │    └─ 콜백 도달 시 시장가 청산
  │
  ├─ 전략 청산 신호 확인
  │    └─ EMA 역교차 / RSI 꺾임 / 거래량 급감 → 시장가 청산
  │
  └─ 해당 없음 → HOLD (다음 캔들까지 대기)
```

### 설정 파라미터

```yaml
# config/strategy/position.yaml 에 추가
exit:
  # 손절
  stop_loss:
    type: atr
    atr_period: 14
    atr_multiplier: 1.5
    min_pct: 0.3
    max_pct: 2.0

  # 익절
  take_profit:
    type: risk_reward
    risk_reward_ratio: 2.0

  # 트레일링 스탑
  trailing_stop:
    activation_r: 1.0            # 1R 수익 시 활성화
    callback_atr_multiplier: 0.8  # ATR × 0.8 콜백

  # 전략 청산
  strategy_exit:
    ema_cross_exit: true          # EMA5/EMA10 역교차
    rsi_reversal_exit: true       # RSI 모멘텀 꺾임
    volume_dry_exit: true         # 거래량 급감
    volume_dry_threshold: 0.3     # 평균 대비 30% 이하

  # 시간 제한
  time_limit:
    max_holding_minutes: 90
    warning_minutes: 75
    force_close_type: market
```

## 영향

- Order Manager: SL/TP 서버사이드 주문 + 트레일링 스탑 클라이언트 관리
- Position Tracker: 포지션별 진입 시간 추적, 90분 타이머
- Notification: 75분 경고, 각 청산 유형별 알림
- 성과 분석: 청산 사유별 통계 (SL/TP/트레일링/전략/시간초과) 기록
