# ADR-009: 첫 번째 스캐너 전략 선택

## 상태

`결정됨`

## 날짜

2026-03-25

## 맥락

시스템의 첫 번째 스캐너 전략을 결정해야 한다.
초기 구현이므로 검증 가능하고 구현 난이도가 적절한 전략이 필요하다.
이 전략으로 전체 파이프라인을 검증하고, 이후 전략을 추가한다.

## 선택지

### 선택지 A: VolumeMomentumScanner (거래량 + 모멘텀)

**설명:**
24시간 거래량 급증과 가격 모멘텀을 결합하여 "움직이기 시작한 종목"을 찾는다.

**스캐닝 로직:**
```
필터 조건:
  1. 24h 거래대금 > 5000만 USDT (유동성 확보)
  2. 24h 거래량 변화율 > 20% (거래량 급증)
  3. 상대 거래량 > 1.5 (20일 평균 대비)
  4. RSI(14) 30~70 (과매수/과매도 극단 제외)
  5. ATR% 0.5~5.0 (적정 변동성)

스코어링:
  거래량 점수 (40%) + 모멘텀 점수 (35%) + 변동성 점수 (25%)
```

**장점:**
- 가장 범용적이고 검증된 필터링 기법
- 유동성 있는 종목만 선정 → 슬리피지 최소화
- 구현 난이도 중간
- 대부분의 Position Strategy와 잘 결합됨

**단점:**
- 이미 움직인 후에 포착할 수 있음 (후행성)
- 뉴스/이벤트에 의한 일시적 급등도 포착 (노이즈)

### 선택지 B: FundingRateScanner (펀딩비 기반)

**설명:**
펀딩비가 극단적인 종목을 찾아 역방향 거래 기회를 포착한다.

**스캐닝 로직:**
```
필터 조건:
  1. 24h 거래대금 > 5000만 USDT
  2. 펀딩비 > +0.01% 또는 < -0.01% (극단적)
  3. 펀딩비 방향이 3회 연속 동일 (지속성 확인)

역방향 논리:
  펀딩비 매우 양수 → 롱이 과다 → SHORT 기회
  펀딩비 매우 음수 → 숏이 과다 → LONG 기회
```

**장점:**
- 선물 시장 고유의 데이터 활용 (차별화)
- 극단적 펀딩비는 가격 반전의 선행 지표가 될 수 있음
- 구현 난이도 낮음 (펀딩비 데이터만 필요)

**단점:**
- 펀딩비가 극단적인 상황 자체가 드묾 → 신호 빈도 낮음
- 강한 추세 시 펀딩비가 계속 극단적일 수 있음 (역추세 위험)
- 단독 사용 시 거래 기회가 부족할 수 있음

### 선택지 C: BreakoutScanner (돌파 임박)

**설명:**
가격이 일정 기간 횡보(수렴)한 후 돌파가 임박한 종목을 찾는다.

**스캐닝 로직:**
```
필터 조건:
  1. 24h 거래대금 > 5000만 USDT
  2. 볼린저 밴드 폭 하위 20% (수축 = 수렴 중)
  3. 최근 N캔들 고가-저가 범위 축소 중
  4. 거래량 감소 후 증가 시작 (Volume Dry-up → Expansion)

스코어링:
  수렴 강도 (50%) + 거래량 패턴 (30%) + 기간 (20%)
```

**장점:**
- 움직이기 "전에" 포착 → 진입 타이밍 유리
- 돌파 시 큰 움직임 기대
- 명확한 진입/손절 기준 (수렴 범위)

**단점:**
- 구현 난이도 높음 (수렴 패턴 판별)
- 가짜 돌파(fake breakout) 빈번
- 수렴이 언제 끝날지 예측 어려움
- 캔들 히스토리가 충분히 필요

### 선택지 D: CompositeScanner (복합 스캐너)

**설명:**
A, B, C를 모두 가벼운 버전으로 구현하고, 각 스캐너의 점수를 가중 합산한다.

**스캐닝 로직:**
```
종합 스코어 =
  거래량·모멘텀 점수 × 0.4
  + 펀딩비 이상 점수 × 0.3
  + 돌파 임박 점수 × 0.3
```

**장점:**
- 단일 기준 편향 제거
- 여러 관점에서 교차 검증

**단점:**
- 초기 구현량이 많음
- 가중치 튜닝 필요
- 개별 전략 검증이 어려움

## 결정

**신규상장 종목 스캐너 (NewListingScanner)** — 기존 선택지에 없는 커스텀 전략.

### 전략 개요

신규상장 종목은 가격 발견(price discovery) 과정에서 변동성과 거래량이 크며, 기술적 분석보다 수급에 의해 움직이는 경향이 있다. 이 특성을 활용한다.

### 스캐닝 로직

```
[후보 풀 구성 - 30개]
  1. Bybit Linear 전체 심볼에서 신규상장 필터 (상장 N일 이내)
  2. 24h 거래대금 최소 기준 충족
  3. 거래량 + 변동성 스코어 상위 30개 선정

[스코어링 - BTC/ETH 추세 반영]
  개별 종목 점수:
    거래량 점수 (30%)
      - 24h 거래대금 순위
      - 상대 거래량 (최근 평균 대비)
    변동성 점수 (25%)
      - ATR% (적정 범위 내에서 높을수록 유리)
      - 볼린저 밴드 폭
    모멘텀 점수 (20%)
      - RSI(14) 위치 (극단 제외, 추세 방향 가점)
      - 가격 변화율 (1h, 4h)
    신규상장 점수 (10%)
      - 상장 경과일 (최신일수록 가점)
    시장 환경 점수 (15%)
      - BTC 추세 정렬 (EMA20 > EMA50 > EMA200 → 강세)
      - ETH 추세 정렬
      - BTC/ETH 추세와 종목 방향 일치 시 가점

  BTC/ETH 추세 반영 방식:
    BTC 강세 + ETH 강세 → 롱 후보 가점, 숏 후보 감점
    BTC 약세 + ETH 약세 → 숏 후보 가점, 롱 후보 감점
    혼조 → 중립 (시장 환경 점수 50점 고정)

[진입 가능 필터]
  스코어링 이후 추가 필터:
    - 이미 포지션 보유 중인 심볼 제외
    - 동일 심볼 최근 손절 이력 있으면 쿨다운 (N시간 대기)
    - 24h 거래대금 급감 중인 종목 제외 (유동성 이탈)
    - 최대 3 포지션 중 빈 슬롯 수만큼만 선정
```

### 설정 파라미터 (scanner.yaml)

```yaml
active: new_listing

strategies:
  new_listing:
    # 신규상장 기준
    listing:
      max_days_since_listed: 30     # 상장 후 최대 30일
      min_days_since_listed: 1      # 상장 당일 제외 (데이터 부족)

    # 최소 유동성
    liquidity:
      min_24h_turnover_usdt: 30000000  # 최소 3000만 USDT

    # 후보 풀
    pool:
      max_candidates: 30

    # 스코어링 가중치
    scoring:
      volume_weight: 0.30
      volatility_weight: 0.25
      momentum_weight: 0.20
      listing_recency_weight: 0.10
      market_env_weight: 0.15
      min_score: 55

    # 진입 필터
    entry_filter:
      cooldown_after_sl_hours: 4    # 손절 후 쿨다운
      min_volume_trend: declining    # 거래량 감소 중이면 제외 (none|declining)

    # BTC/ETH 추세 판단
    market_env:
      ema_periods: [20, 50, 200]
      timeframe: 1h                  # 추세 판단 타임프레임
```

## 영향

- Data Collector: BTC/ETH 캔들 상시 수집 + `instruments-info` API로 상장일 조회
- 새로운 데이터 필요: 심볼별 상장일 (`launchTime`)
- Position Strategy: BTC/ETH 추세 데이터를 공유받아 방향 필터로 활용 가능
- 스캐너가 "롱 유리 / 숏 유리" 힌트를 함께 전달할 수 있음
