# ADR-010: 스캐너 실행 주기 및 선정 개수

## 상태

`결정됨`

## 날짜

2026-03-25

## 맥락

스캐너의 실행 주기와 한 번에 선정하는 최대 종목 수를 결정해야 한다.
이 값은 거래 빈도, 자본 분산, 시스템 부하에 직접 영향을 준다.

## 선택지

### 실행 주기

#### A: 5분

**장점:** 시장 변화에 빠르게 대응, 기회 포착률 높음
**단점:** 종목이 자주 바뀌어 불안정, Position Strategy에 충분한 분석 시간 부족

#### B: 15분

**장점:** 캔들 타임프레임(15m)과 동기화 가능, 적절한 반응 속도
**단점:** 급변 시 대응 지연

#### C: 1시간

**장점:** 안정적인 종목 유지, P2 부하 최소, 노이즈 필터링
**단점:** 기회 포착 지연

### 선정 개수

#### X: 3개

**장점:** 집중 투자, 자본 분산 최소, 모니터링 용이
**단점:** 기회 제한

#### Y: 5개

**장점:** 적절한 분산, 다양한 기회 포착, 자산관리 전략(max_concurrent_positions: 3)과 균형
**단점:** 포지션당 자본 감소

#### Z: 10개

**장점:** 기회 최대화, 높은 분산
**단점:** 관리 복잡, 소액 자본 시 포지션 크기 너무 작아짐, P2 부하 증가

## 결정

### 실행 주기: 온디맨드 (이벤트 기반)

고정 주기가 아닌, **포지션 빈 슬롯이 생길 때만** 스캐너를 실행한다.

```
스캐너 트리거 조건:
  1. 시스템 시작 시 (현재 포지션 < 3)
  2. 포지션 청산 완료 시 (빈 슬롯 발생)
  3. 위 조건에서 스캔했으나 진입 조건 불충족 시 → 일정 대기 후 재스캔

재스캔 대기: 15분 (primary 타임프레임과 동기화)
```

**이유:**
- 최대 3 포지션이므로 슬롯이 빌 때만 새 종목이 필요하다
- 불필요한 주기적 스캔은 리소스 낭비이며 종목이 불안정하게 변경될 수 있다
- 포지션 청산 이벤트가 자연스러운 트리거가 된다

### 선정 개수 및 자본 배분

```
후보 풀:        30개 (REST 1차 필터)
최대 포지션:    3개 동시 운영
종목당 자본:    전체 자본의 5% (15% 투입, 85% 유보)
스캔 시 선정:   빈 슬롯 수만큼 (1~3개)
```

### 선정 개수 vs 동시 포지션

```
후보 풀 30개 → 스코어링 → 상위 N개 선정 (N = 빈 슬롯 수)
  → Position Strategy가 진입 조건 평가
  → 조건 충족 시 진입 (종목당 5%)
  → 미충족 시 대기, 15분 후 재스캔
```

## 영향

- 스캐너 실행이 이벤트 기반 → P3에서 P2로 "슬롯 발생" 메시지 필요
- 고정 interval 설정 불필요 → `scanner_interval_sec` 제거, `rescan_delay_sec: 900` 추가
- 최대 3 포지션 × 5% = 15% 자본 사용 → 보수적 운영
- Position Strategy는 스캐너가 선정한 종목만 즉시 평가 (대기 목록 없음)
