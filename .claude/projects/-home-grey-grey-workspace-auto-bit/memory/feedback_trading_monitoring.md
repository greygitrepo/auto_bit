재시작 후 20분 이내 거래가 없으면 파라미터나 전략에 문제가 있는 것으로 판단한다.

**Why:** 사용자는 빠른 피드백 루프를 중시함. 멈춰있는 시스템을 수동적으로 기다리는 것은 시간 낭비.

**How to apply:**
1. 시스템 재시작 후 반드시 거래 발생 여부를 확인
2. 20분 무거래 시 → 즉시 로그 분석 → 원인 파악 → 파라미터/코드 수정 → 재시작 → 재테스트
3. 목표는 단타 안정화 및 수익성
4. "문제파악 → 수정 → 재테스트" 사이클을 능동적으로 돌릴 것
